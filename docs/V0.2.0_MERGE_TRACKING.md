## Future Feature: Global Knowledge Graph Index (`.mnemglobal`)

### Overview

A centralized global index that tracks all mnem knowledge graphs on a device, enabling agents and humans to discover, manage, and cross-reference all local knowledge graphs from a single location.

### Problem Statement

Currently, each project/repo has its own isolated `.mnem/` folder. This means:
- Agents can't discover all knowledge graphs on the device
- No unified way to manage multiple graphs
- Hard to get an overview of all knowledge stored across projects
- Cross-referencing between graphs requires manual configuration

### Proposed Solution: `.mnemglobal`

**Location:** `~/.mnemglobal/` (home directory, platform-appropriate)

```
~/.mnemglobal/
├── config.toml              # Global configuration
├── index.json               # Registry of all knowledge graphs
├── graph-meta/              # Metadata for each graph
│   ├── {uuid}.meta          # Metadata files for each graph
│   └── ...
├── shared/                  # Optional shared embeddings/cache
│   └── ...
└── logs/                    # Global mnem logs
```

### Core Architecture

#### 1. Index File (`index.json`)

```json
{
  "version": "1.0",
  "created": "2026-04-30T12:00:00Z",
  "graphs": [
    {
      "id": "abc123",
      "name": "work-project",
      "path": "/home/user/code/myproject/.mnem",
      "alias": "work",
      "description": "Work-related knowledge",
      "tags": ["work", "project", "code"],
      "created": "2026-01-15T10:00:00Z",
      "lastAccessed": "2026-04-30T08:30:00Z",
      "nodeCount": 1523,
      "isActive": true
    },
    {
      "id": "def456",
      "name": "personal-notes",
      "path": "/home/user/notes/.mnem",
      "alias": "notes",
      "description": "Personal notes and thoughts",
      "tags": ["personal", "notes"],
      "created": "2026-02-20T14:00:00Z",
      "lastAccessed": "2026-04-29T22:00:00Z",
      "nodeCount": 3421,
      "isActive": true
    }
  ]
}
```

#### 2. Graph Registration

When `mnem init` is run in a new directory:
1. Create `.mnem/` in that directory (as before)
2. Automatically register with `.mnemglobal/index.json`
3. Generate unique ID and optional alias
4. Store metadata in `graph-meta/{id}.meta`

#### 3. Reference Mechanism

Each `.mnem/config.toml` can optionally contain:
```toml
[global]
mnemglobal = "~/.mnemglobal"  # Path to global index
autoRegister = true           # Auto-register with global index
```

### Commands (New)

```bash
# Initialize global index
mnem global init

# List all knowledge graphs
mnem global list
# Output:
# +-------------+-------------+---------------+--------+
# | Alias       | Name        | Path          | Nodes  |
# +-------------+-------------+---------------+--------+
# | work        | work-project| ~/code/myproj | 1,523  |
# | notes       | personal    | ~/notes       | 3,421  |
# +-------------+-------------+---------------+--------+

# Show detailed info about a graph
mnem global info work

# Add alias for a graph
mnem global alias work "work-project"

# Tag a graph
mnem global tag work "project,client-x,important"

# Search across all graphs
mnem global search "machine learning"

# Import a graph reference (from external location)
mnem global import /external/path/.mnem --alias "archive"

# Remove from global index (doesn't delete actual data)
mnem global remove work

# Check global index health
mnem global doctor

# Show global statistics
mnem global stats
```

### TUI for Global Management

```bash
mnem global tui
```

Opens an interactive TUI showing:
- All knowledge graphs in a table
- Quick stats per graph
- Ability to tag, alias, search
- Cross-graph search results
- Health status of each graph

### Agent Integration

Agents can use the global index to:

1. **Discover all knowledge**:
   ```python

   # Find relevant graphs by tags, name, description
   ```

2. **Cross-graph retrieval**:
   ```python
   # Query multiple graphs in parallel
   # Aggregate results with source attribution
   ```

3. **Automatic context loading**:
   ```bash
   mnem retrieve "project context" --global --tags "work"
   ```

### Implementation Priority

| Phase | Feature | Description |
|-------|---------|-------------|
| 1 | Basic Index | Create `~/.mnemglobal/` with `index.json` |
| 2 | Auto-registration | Auto-register new graphs |
| 3 | CLI Commands | `mnem global list/info/tag` |
| 4 | Cross-graph Search | Search across all graphs |
| 5 | TUI | Interactive management UI |
| 6 | Agent Tools | MCP tools for global discovery |

### Potential Drawbacks & Solutions

| Drawback | Solution |
|----------|----------|
| **Orphaned refs** | `mnem global doctor` detects moved/deleted repos |
| **Privacy** | Agent must have read access to `~/.mnemglobal` - user controls |
| **Performance** | Index is small JSON, lazy-load full metadata |
| **Conflict of interest** | Tags help disambiguate graphs with similar names |
| **Data leakage** | Agent can't see graphs unless explicitly registered |

### Migration Path

1. First run of any mnem command checks for `~/.mnemglobal/`
2. If not exists, prompt to initialize with `mnem global init`
3. Existing `.mnem/` repos can be added with `mnem global import /path/to/.mnem`
4. Backward compatible: graphs work without global index

### Notes for Implementation

- Keep index file lean (only UUID, path, alias, tags)
- Heavy metadata in separate files under `graph-meta/`
- Use file locking for concurrent access
- Consider SQLite for index if JSON becomes too large
- TUI using same framework as `mnem integrate`
